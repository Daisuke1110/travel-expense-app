# アプリ開発手順（MVP向け）

Travel Expense App を実務エンジニア風に進めるための標準プロセス（MVP版）。

---

# ① 目標・ゴール定義 (What/Why)
- 何のため・誰のため（海外旅行中の支出管理、本人＋友人）
- MVPでやる/やらないを決める → product.md に反映

---

# ② 要件定義（機能 / 非機能）
- 機能例: トリップ作成/一覧/詳細、支出登録/表示/削除、円換算
- 非機能例: パフォーマンス(主要画面 ~3秒)、セキュリティ(MVPは簡易認証)、コスト(月数百円目安)、オフライン方針(MVPは静的キャッシュのみ)
- requirements.md にまとめる

---

# ③ 全体設計（アーキテクチャ）
- PWA + AWSサーバレス（React/Vite, FastAPI Lambda, DynamoDB, S3+CloudFront）
- 認証: MVPはヘッダ`X-Debug-User-Id`、将来 Cognito
- architecture.md に 1ページで整理

---

# ④ 詳細設計（ドキュメント群）
- data-model.md（スキーマ・GSI方針）
- backend.md（API仕様: /me/trips, /trips, /trips/{id}, /expenses系, レート編集, 支出編集/削除, トリップ削除）
- frontend.md（画面仕様・オフライン文言・権限制御表示）
- infra.md（Terraform方針・CORS・ACM ARN・秘匿情報扱い）

---

# ⑤ プロジェクト準備
- リポジトリ/ブランチ戦略（main + feature/*）
- ディレクトリ構成: backend/, frontend/, infra/terraform/, docs/
- Issue でタスク管理

---

# ⑥ 実装（縦に小さく進める）
1. Backend: `GET /me/trips` → `GET /trips/{id}` → `GET /trips/{id}/expenses`（ローカルDynamoDB or モック）
2. Frontend: Home → TripDetail をモックで繋ぎ、実APIに差し替え
3. Backend: `POST /trips/{id}/expenses`（金額は整数、currencyはbase通貨固定、UTC）
4. Frontend: 支出追加モーダル/バリデーション
5. Backend: 認可チェック（TripMembers）を全APIに適用
6. Backend: `PATCH /expenses/{expenseId}` と `DELETE /trips/{tripId}`（必要なら後ろのスプリントに分離）

---

# ⑦ テスト（最小スモーク）
- Backend: ユニット＋APIスモーク（/me/trips, /trips/{id}, /expenses系）
- Frontend: 画面遷移、フォームバリデーション、APIエラー時のオフライン文言表示
- 簡易 test-plan を docs/ にメモ

---

# ⑧ レビュー・修正
- 自己レビュー／コード整形
- UI/UX チェック
- フィードバックをIssue化して反映

---

# ⑨ デプロイ（dev→prod）
- Terraform 最小セット: API Gateway, Lambda, DynamoDB, S3, CloudFront, Route53, ACM
- dev は手動デプロイから開始（CI/CDは後から）
- prod は少人数に限定したソフトローンチ

---

# ⑩ 運用・改善
- CloudWatch Logs でエラー確認
- コスト確認（Billing / Cost Explorer）
- 追加機能の検討（OCR/割り勘/タグ/グラフ等）をIssue化し、段階的に実装
